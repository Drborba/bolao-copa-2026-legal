2. DADOS COLETADOS 2.1 Dados Fornecidos Diretamente pelo Usuário 2.1.1 Cadastro Tradicional (Email e Senha) Durante o registro manual, coletamos:

Email: Para autenticação, recuperação de senha e comunicações essenciais; Senha: Armazenada de forma criptografada (hash bcrypt/argon2) – nunca em texto simples; Nickname/Username: Nome de exibição nos rankings e bolões (obrigatório); Nome Completo: Opcional, para personalização do perfil.

2.1.2 Login Social (Google/Facebook) Ao optar por autenticação via redes sociais, coletamos apenas os dados autorizados por você:

Google: Email, nome, foto de perfil do Google; Facebook: Email, nome, foto de perfil do Facebook.

Nota Importante: Não acessamos suas mensagens, contatos, posts ou outras informações privadas das redes sociais. A coleta é limitada ao escopo de autenticação OAuth 2.0. 2.1.3 Dados de Perfil Você pode fornecer voluntariamente: Foto de Perfil (Avatar): Imagem enviada para personalização, armazenada em nosso servidor (Supabase Storage); Preferências de Notificação: Configurações sobre quais alertas deseja receber. 2.1.4 Dados de Uso do Aplicativo Durante a utilização do Serviço, coletamos: Palpites de Jogos: Placares previstos para partidas de futebol; Bolões Criados/Participados: Nomes dos bolões, códigos de convite, posição no ranking; Estatísticas de Desempenho: Pontuação total, cravadas (placares exatos), acertos de vencedor; Palpites de Ouro: Marcações de palpites multiplicadores (2x pontos); Interações Sociais: Convites enviados, participantes adicionados/removidos. 2.2 Dados Coletados Automaticamente 2.2.1 Dados Técnicos e de Dispositivo Identificador de Dispositivo: ID único gerado pelo sistema operacional (usado para push notifications); Modelo e Versão do Dispositivo: Ex: iPhone 13, Android 12 (para compatibilidade e suporte); Sistema Operacional: iOS ou Android (versão); Versão do Aplicativo: Para análise de crashes e atualizações; Endereço IP: Coletado temporariamente para segurança e prevenção de fraudes; Estado de Conexão de Rede: Se está conectado via WiFi ou dados móveis (para otimizar sincronização). 2.2.2 Tokens de Autenticação e Sessão JWT (JSON Web Token): Token de acesso com validade de 1 hora, renovado automaticamente; Refresh Token: Válido por 30 dias, para renovação de sessão; Push Notification Token: Identificador único do Expo Notifications para envio de alertas. 2.2.3 Logs de Erro e Desempenho Coletamos automaticamente: Logs de Erro (Crash Reports): Informações sobre falhas do Aplicativo (stack traces, linha de código com erro); Logs de Depuração: Registros de ações do usuário para diagnóstico de problemas (ex: "Usuário tentou fazer palpite após deadline"). ⚠️ ATENÇÃO – POSSÍVEL VIOLAÇÃO DE PRIVACIDADE: Os logs de depuração podem conter dados sensíveis como email, IDs de usuário ou mensagens de erro detalhadas. É CRÍTICO implementar sanitização de logs antes do envio para produção. Recomenda-se: Remover dados pessoais dos logs (emails, nomes, IDs visíveis); Usar IDs anonimizados para rastreamento de erros; Implementar ferramenta de crash reporting (Sentry, Crashlytics) com filtragem de PII (Personally Identifiable Information). 2.2.4 Dados de Localização (LIMITADO) O Aplicativo NÃO coleta localização geográfica precisa via GPS. Podemos inferir localização aproximada através de: Endereço IP (nível de cidade/região); Configuração de idioma/fuso horário do dispositivo. Finalidade: Exibir horários de jogos no fuso local, recomendar campeonatos regionais (futuro). 2.3 Dados de Terceiros (Integrações) 2.3.1 Dados de Partidas de Futebol Obtemos informações de jogos (times, placares, horários) de APIs públicas ou licenciadas (ex: API-Football). Esses dados são públicos e não identificam usuários. 2.3.2 Publicidade (AdMob – Futuro) Quando implementado, o Google AdMob poderá coletar: ID de Publicidade (GAID/IDFA): Identificador usado para anúncios personalizados; Impressões e Cliques: Métricas de interação com anúncios; Dados de Navegação: Páginas visualizadas no Aplicativo (anonimizados). Você pode desabilitar anúncios personalizados nas configurações do seu dispositivo (iOS: "Rastreamento Limitado de Anúncios" / Android: "Desativar Personalização de Anúncios"). 3. COMPARTILHAMENTO DE DADOS COM TERCEIROS 4.1 Prestadores de Serviço (Subprocessadores) Compartilhamos dados com os seguintes terceiros exclusivamente para operação do Aplicativo: 4.1.1 Supabase (Infraestrutura de Backend) Dados Compartilhados: Todos os dados de usuário (email, senha hash, perfil, palpites, bolões); Finalidade: Hospedagem de banco de dados, autenticação, armazenamento de arquivos (avatares); Localização dos Servidores: Estados Unidos/Europa (Supabase usa AWS); Segurança: Criptografia em trânsito (TLS 1.3) e em repouso (AES-256); Política de Privacidade: https://supabase.com/privacy ⚠️ ATENÇÃO – TRANSFERÊNCIA INTERNACIONAL DE DADOS: Como o Supabase hospeda dados fora do Brasil, há transferência internacional. A LGPD exige que esta transferência seja feita para países com nível adequado de proteção (Art. 33) ou mediante cláusulas contratuais padrão. Verifique se o Supabase é certificado (ex: Privacy Shield successor) ou se há Contrato de Controlador-Processador assinado. 4.1.2 Expo (Notificações Push) Dados Compartilhados: Push notification token (ID único do dispositivo); Finalidade: Envio de notificações push (lembretes, resultados); Política de Privacidade: https://expo.dev/privacy 4.1.3 Google (OAuth e AdMob – Futuro) OAuth (Login com Google): Dados Compartilhados: Email, nome, foto de perfil (com seu consentimento); Finalidade: Autenticação facilitada; Política: https://policies.google.com/privacy AdMob (Publicidade): Dados Compartilhados: ID de publicidade, impressões de anúncios; Finalidade: Exibição de anúncios e geração de receita; Controle: Você pode desativar anúncios personalizados nas configurações do dispositivo. 4.1.4 Facebook (OAuth – Opcional) Dados Compartilhados: Email, nome, foto de perfil (com seu consentimento); Finalidade: Autenticação via Facebook; Política: https://www.facebook.com/privacy/explanation 4.2 Compartilhamento por Obrigação Legal Podemos divulgar seus dados se exigido por: Ordem judicial ou requisição de autoridade competente (Polícia, Ministério Público, ANPD); Cumprimento de lei, regulamento ou processo legal; Proteção de direitos, propriedade ou segurança do Aplicativo, usuários ou terceiros. Em tais casos, limitaremos a divulgação ao estritamente necessário e notificaremos você, salvo se proibido por lei. 4.3 NÃO Vendemos Seus Dados Nunca vendemos, alugamos ou comercializamos seus dados pessoais para terceiros. 5. ARMAZENAMENTO E SEGURANÇA DOS DADOS 5.1 Onde Seus Dados São Armazenados Banco de Dados: Supabase (PostgreSQL), servidores localizados nos EUA/Europa; Armazenamento de Arquivos (Avatares): Supabase Storage (AWS S3); Local (Dispositivo): AsyncStorage (preferências de tema, tokens de sessão - não criptografado). ⚠️ ATENÇÃO – AsyncStorage NÃO É SEGURO: Dados armazenados localmente via AsyncStorage (como tokens de sessão) não são criptografados. Embora tokens JWT expirem em 1 hora, há risco teórico de extração por malware em dispositivos rooteados/jailbreak. Considere implementar criptografia local via expo-secure-store para tokens sensíveis. 5.2 Prazo de Retenção Conta Ativa: Mantemos dados enquanto sua conta estiver ativa; Conta Inativa: Após 365 dias sem login, podemos anonimizar ou excluir dados, mediante notificação prévia; Exclusão de Conta: Após solicitação de exclusão, removemos dados em até 30 dias, exceto: Dados necessários para cumprimento de obrigações legais (ex: registros de transações para fins fiscais - se aplicável); Estatísticas anonimizadas em bolões compartilhados (para preservar integridade das competições). 5.3 Medidas de Segurança Implementamos controles técnicos e organizacionais para proteger seus dados: Medidas Técnicas: Criptografia em Trânsito: HTTPS/TLS 1.3 em todas as comunicações; Criptografia em Repouso: Dados armazenados no Supabase são criptografados (AES-256); Hash de Senhas: Senhas protegidas com algoritmo bcrypt (custo fator 12+); Autenticação de Dois Fatores (2FA): Planejado para versões futuras; Row-Level Security (RLS): Políticas de acesso granular no banco de dados (usuários só acessam próprios dados); Tokens JWT com Expiração: Tokens de acesso expiram em 1 hora; refresh tokens em 30 dias; Rate Limiting: Proteção contra ataques de força bruta em login. Medidas Organizacionais: Acesso a dados restrito a pessoal autorizado com necessidade de conhecimento (need-to-know); Treinamento de equipe sobre boas práticas de privacidade e segurança; Auditorias periódicas de segurança e testes de vulnerabilidade (planejado). Backup e Recuperação: Backups automáticos diários do banco de dados (Supabase); Plano de recuperação de desastres (DRP) para restauração em até 24 horas. ⚠️ Nenhum sistema é 100% seguro. Encorajamos você a: Usar senha forte e única; Não compartilhar suas credenciais; Ativar autenticação de dois fatores quando disponível; Manter seu dispositivo atualizado e protegido. 6. COOKIES E TECNOLOGIAS DE RASTREAMENTO 6.1 Tecnologias Utilizadas O Aplicativo não utiliza cookies tradicionais de navegador, mas emprega tecnologias similares: 6.1.1 Armazenamento Local (AsyncStorage) Tipo: Key-value storage no dispositivo; Dados Armazenados: Tokens de autenticação (JWT + Refresh Token); Preferências de tema (modo claro/escuro); Configurações de notificações; Estado de onboarding (se usuário já viu tutorial). Finalidade: Manter sessão ativa e configurações personalizadas; Validade: Persistente até logout ou desinstalação do app. Gestão: Você pode limpar esses dados fazendo logout ou desinstalando o Aplicativo. 6.1.2 Tokens de Sessão (JWT) Tipo: JSON Web Token (formato Base64); Conteúdo: ID do usuário, email, data de emissão/expiração; Validade: 1 hora (access token) / 30 dias (refresh token); Finalidade: Autenticação segura sem necessidade de reenviar senha a cada requisição; Segurança: Assinados com chave secreta (HMAC-SHA256); não podem ser alterados sem detecção. 6.1.3 Push Notification Tokens Tipo: Expo Push Token (identificador único do dispositivo); Validade: Persistente até desinstalação ou revogação manual; Finalidade: Envio de notificações push direcionadas; Compartilhamento: Token compartilhado com Expo para entrega de notificações. Gestão: Você pode desabilitar notificações em "Configurações" → "Notificações" ou nas configurações do dispositivo. 6.2 Rastreamento de Anúncios (AdMob – Futuro) Quando implementado, o Google AdMob utilizará: ID de Publicidade (GAID/IDFA): Identificador anônimo para rastreamento de anúncios; Cookies de Terceiros: Anunciantes podem usar cookies para medir eficácia de campanhas. Controle: iOS: Configurações → Privacidade → Rastreamento → Desativar "Permitir que apps solicitem rastreamento"; Android: Configurações → Google → Anúncios → Desativar personalização de anúncios. Nota: Desabilitar anúncios personalizados não reduz o número de anúncios, mas torna-os genéricos (não baseados em seu perfil). 6.3 Não Rastreamos Você Entre Aplicativos O Aplicativo não implementa rastreamento cross-app (seguir sua atividade em outros apps ou sites). Dados coletados são usados exclusivamente dentro do Bolão Copa 2026. 7. DIREITOS DO TITULAR DE DADOS (LGPD) Em conformidade com a Lei Geral de Proteção de Dados (LGPD), você possui os seguintes direitos: 7.1 Direito de Acesso (Art. 18, II) Você pode solicitar confirmação sobre quais dados pessoais mantemos sobre você e obter cópia desses dados. Como exercer: Envie email para [email de suporte] com assunto "Solicitação de Acesso a Dados – LGPD". 7.2 Direito de Retificação (Art. 18, III) Você pode corrigir dados incompletos, inexatos ou desatualizados. Como exercer: Direto no App: Menu "Configurações" → "Editar Perfil" (para nickname, nome, avatar); Por Email: Para correção de email de cadastro (requer validação de identidade). 7.3 Direito de Exclusão (Art. 18, VI) Você pode solicitar a eliminação de dados pessoais tratados com base em seu consentimento ou quando não mais necessários. Como exercer: Direto no App: Menu "Configurações" → "Excluir Conta" (se implementado); Por Email: Envie solicitação para [email de suporte] com assunto "Exclusão de Conta – LGPD". ⚠️ CRÍTICO – FUNCIONALIDADE DE EXCLUSÃO DE CONTA NÃO IMPLEMENTADA: O código analisado NÃO possui funcionalidade de "Deletar Conta" visível. Isso é uma violação crítica da LGPD (Art. 18, VI). É IMPERATIVO implementar: Botão "Excluir Conta" em Configurações; Confirmação dupla (prevenir exclusões acidentais); Remoção automática de dados em até 30 dias; Notificação de confirmação de exclusão. Limitações à Exclusão: Dados anonimizados em rankings de bolões encerrados (para preservar integridade histórica); Logs de acesso (IP) mantidos por até 6 meses para fins de segurança (Art. 11 do Marco Civil); Informações necessárias para cumprimento de obrigações legais. 7.4 Direito de Portabilidade (Art. 18, V) Você pode solicitar a transferência de seus dados para outro fornecedor de serviço, em formato estruturado e interoperável. Como exercer: Envie email para [email de suporte]. Forneceremos dados em formato JSON ou CSV. 7.5 Direito de Oposição (Art. 18, § 2º) Você pode opor-se ao tratamento de dados realizado com base em legítimo interesse. Exemplo: Opor-se ao uso de dados para análises de uso ou publicidade. Como exercer: Entre em contato via [email de suporte]. Avaliaremos se há motivos legítimos que justifiquem a manutenção do tratamento. 7.6 Direito de Revogação de Consentimento (Art. 8º, § 5º) Você pode revogar o consentimento dado para tratamento de dados, sem afetar a licitude do tratamento realizado anteriormente. Como exercer: Notificações Push: Desabilite em "Configurações" → "Notificações"; Login Social: Revogue acesso em Google Account ou Facebook Apps; AdMob: Desative anúncios personalizados nas configurações do dispositivo. 7.7 Direito de Informação sobre Compartilhamento (Art. 18, VII) Você pode solicitar informações sobre com quem compartilhamos seus dados. Resposta: Consulte a seção "4. Compartilhamento de Dados com Terceiros" desta Política. Para dúvidas adicionais, contate [email de suporte]. 7.8 Direito de Revisão de Decisões Automatizadas (Art. 20) O Aplicativo não utiliza decisões automatizadas que afetem significativamente seus direitos (ex: perfis de crédito, discriminação algorítmica). O sistema de pontuação é baseado em regras fixas e transparentes (Índice 3.0). 7.9 Prazos para Resposta Responderemos às solicitações em até 15 dias corridos, podendo ser prorrogado por mais 15 dias mediante justificativa. Gratuidade: Atendimento de direitos LGPD é gratuito. Podemos cobrar taxa razoável apenas se solicitações forem manifestamente infundadas ou excessivas. 8. GLOSSÁRIO DE TERMOS Para facilitar a compreensão, definimos: Dado Pessoal: Informação relacionada a pessoa natural identificada ou identificável (ex: nome, email, IP). Dado Sensível: Dados sobre origem racial/étnica, religião, opinião política, saúde, vida sexual, etc. (Art. 5º, II da LGPD) – NÃO coletamos dados sensíveis. Tratamento: Qualquer operação com dados pessoais (coleta, armazenamento, uso, compartilhamento, exclusão). Controlador: Entidade que toma decisões sobre tratamento de dados (nós, [Nome da Empresa/Desenvolvedor]). Operador: Entidade que trata dados em nome do Controlador (ex: Supabase). Titular: Você, pessoa natural a quem os dados pessoais se referem. Anonimização: Processo de tornar dados não identificáveis de forma irreversível. Consentimento: Manifestação livre, informada e inequívoca pela qual o titular concorda com o tratamento de seus dados.
